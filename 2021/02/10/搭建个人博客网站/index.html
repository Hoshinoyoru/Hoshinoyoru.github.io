搭建个人博客网站

前言：今天我搭建了自己的第一个个人博客网站。本来我只是想用它来堆放自己的文章的，但是文章的编写都是使用的Markdown格式，这和我原来想象的可以用纯文本格式编辑有点不一样，不过好在Typora用来写小说体验也不错（就是格式转换比较麻烦，我一般在Typora上写完了再剪切到Word上排版）。既然文章可以写Markdown格式，我就顺便记录一些平时自己折腾电脑的过程写成博客，虽然没有什么技术含量（并且我以后也不打算从事IT），但是写出来还是蛮有成就感。于是我就画十分钟大概了解了Markdown语法，详细记录了我搭建网站时踩的坑。

了解Hexo

这里引用一下Hexo官网的原话：

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

（Markdown竟然是渲染引擎……）

环境搭建

在安装Hexo之前，需要安装Node.js和Git(Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)。

简单介绍一下Node.js和Git

引用菜鸟教程上的介绍：

简单的说 Node.js 就是运行在服务端的 JavaScript。
Node.js 是一个基于Chrome JavaScript 运行时建立的一个平台。
Node.js是一个事件驱动I/O服务端JavaScript环境，基于Google的V8引擎，V8引擎执行Javascript的速度非常快，性能非常好。

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。
Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。
Git 与常用的版本控制工具 CVS, Subversion 等不同，它采用了分布式版本库的方式，不必服务器端软件支持。

由于新版的nodejs已经集成了npm，所以之前npm也一并安装好了。同样可以通过输入 “npm -v” 来测试是否成功安装。

然后我翻看了Hexo的官方文档。

可以使用下面这条指令来安装Hexo
npm install -g hexo-cli
对于熟悉 npm 的进阶用户，可以仅局部安装 hexo 包。
npm install hexo
安装以后，可以使用以下两种方式执行 Hexo：
1.npx hexo <command>
2.将 Hexo 所在的目录下的 node_modules 添加到环境变量之中即可直接使用 hexo <command>：
echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile

执行Hexo的意思大概是和上面那篇知乎文章是一个意思，但是执行Hexo的时候直接在Bash中输入hexo <command>就可以了。
