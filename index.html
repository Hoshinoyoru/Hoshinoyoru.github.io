搭建个人博客网站

前言：今天我搭建了自己的第一个个人博客网站。本来我只是想用它来堆放自己的文章的，但是文章的编写都是使用的Markdown格式，这和我原来想象的可以用纯文本格式编辑有点不一样，不过好在Typora用来写小说体验也不错（就是格式转换比较麻烦，我一般在Typora上写完了再剪切到Word上排版）。既然文章可以写Markdown格式，我就顺便记录一些平时自己折腾电脑的过程写成博客，虽然没有什么技术含量（并且我以后也不打算从事IT），但是写出来还是蛮有成就感。于是我就花十分钟大概了解了Markdown语法，详细记录了我搭建网站时踩的坑。

了解Hexo

这里引用一下Hexo官网的原话：

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Hexo的安装

我翻看了Hexo的官方文档。

可以使用下面这条指令来安装Hexo
npm install -g hexo-cli
对于熟悉 npm 的进阶用户，可以仅局部安装 hexo 包。
npm install hexo
安装以后，可以使用以下两种方式执行 Hexo：
1.npx hexo <command>
2.将 Hexo 所在的目录下的 node_modules 添加到环境变量之中即可直接使用 hexo <command>：
echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile

执行Hexo的意思大概是和上面那篇知乎文章是一个意思，但是执行Hexo的时候直接在Bash中输入hexo <command>就可以了。

建站

做多几次后我发现建站和建仓库，生成SSH没有前后之分，我就按照官方文档的顺序来写了。首先我们需要一个文件来储存博客网站所需要的文件。可以在你准备建立的文件夹的根目录空白地区单击右键再Git Bash Here，依次执行
$ hexo init <folder>
$ cd <folder>
$ npm install
命令（是你建立的文件夹的名字），就会在该目录下生成一个文件夹。或者你自己新建一个文件夹，然后右键单击文件夹再Bash，执行
$ hexo init
$ npm install
就不用建立文件夹后再Cd到那个文件夹下面去。

在文件夹中有几个需要注意的文件：
_config.yml(网站的配置信息,参数含义可参考官方文档)
source（存放用户资源的地方，一般是文章的Md格式）
themes（主题文件夹，更改主题会用到）

需要注意的是

文件夹创建好了之后，配置文件中的Deployment的参数只有type，需要改为：

type: git repository: https://github.com/Username/Username.github.io.git branch: main

（网上的教程大多数都是Master，但是Hexo更新快，已经变成了Main，这是个大坑。并且每个参数的冒号与参数值之间有一个空格,Isername是Github账号的用户名）

创建Github的仓库

没啥好说，多上网冲冲浪吧，这玩意也不需要挂梯子。需要注意，仓库名字的格式必须是Username.Github.io，这样当你在地址栏中输入你的网站的网址（就是这个Username.Github.io）才会被搜索到。

其他的，我们生成的网页是托管在Github上的，起到服务器作用的是Github Pages。
